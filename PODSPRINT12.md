# Sprint 12 — External Validation

**Okres**: 15-21 czerwca 2026
**Cel sprintu**: Prawdziwi uzytkownicy korzystaja z ServeIQ. Zbieranie dowodow wartosci.

---

## Taski

### A1. Kick-off pilota
- [ ] Spotkanie kick-off z klubem pilotowym (video/osobiste)
- [ ] Walk-through dla trenerow: dashboard, aktywnosci, notatki, przeglady
- [ ] Walk-through dla koordynatora: dashboard klubu, grupy, metryki
- [ ] Wyslanie zaproszen do rodzicow z instrukja
- [ ] Ustalenie cotygodniowego check-inu (piatek 10 min)

### A2. Rapid bug fixing
**Regula: tylko to co blokuje realne uzycie.**
- [ ] Codzienne sprawdzanie feedbacku (formularz + email + check-in)
- [ ] Fix krytycznych w < 24h
- [ ] Fix waznych w < 48h
- [ ] Zero nowych features — tylko naprawy i drobne poprawki
- [ ] Deploy codziennie jesli sa fixy

### A3. Monitoring uzycia
**Codziennie sprawdzac (z analytics z Sprint 11):**
- [ ] Kto sie loguije? (trenerzy, rodzice, admin)
- [ ] Co robia? (aktywnosci, notatki, przeglady)
- [ ] Gdzie odpadaja? (ktore ekrany maja bounce)
- [ ] Co jest uzywane najczesciej?
- [ ] Co jest pomijane?

**Tygodniowy raport:**
- Aktywni uzytkownicy per rola
- Top akcje
- Friction points
- Feedback podsumowanie

### A4. Zbieranie dowodow wartosci
- [ ] Cytaty od trenerow: "Co Ci sie podoba?", "Co zmienilo sie w pracy?"
- [ ] Cytaty od rodzicow: "Czy widzisz wartosc?", "Co jest najwazniejsze?"
- [ ] Screenshots uzycia (za zgoda)
- [ ] Metryki ilosciowe:
  - Ile aktywnosci zaplanowano?
  - Ile obserwacji dodano?
  - Ile przegladow opublikowano?
  - Ilu rodzicow sprawdzilo profil dziecka?
- [ ] NPS lub prosta ankieta: "Czy polecilbys ServeIQ innemu klubowi?" (1-10)

### A5. Iteracja na podstawie feedbacku
- [ ] Zebrany feedback -> podział na:
  - **Must-fix now**: blokuje uzycie
  - **Next sprint**: wazne ale nie blokujace
  - **Backlog**: nice-to-have
- [ ] Fix must-fix immediately
- [ ] Przygotowanie backlogu na nastepna faze

## Definition of Done

- [ ] Min. 1 klub aktywnie korzysta (trenerzy loguja sie, dodaja dane)
- [ ] Min. 5 rodzicow sprawdzilo profil dziecka
- [ ] Krytyczne bledy naprawione w < 24h
- [ ] Zebrano min. 3 cytaty/opinie od uzytkownikow
- [ ] Dane uzycia udokumentowane (tygodniowy raport)
- [ ] Feedback skategoryzowany (must-fix / next / backlog)
